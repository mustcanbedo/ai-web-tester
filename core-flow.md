## 流程一：用户登录与注册

**前置条件**：浏览器已清除 `localStorage` 中的 `token` 和 `user`。

> ⚠️ **已登录检测**：测试系统可能自动恢复了上次的登录态（cookies）。打开首页后，先检查导航栏右侧：
> - 如果看到 **"登录/注册"** 按钮（role=`button`，name 包含"登录"）→ 未登录，继续下面的步骤
> - 如果看到用户名或手机号后四位（如 `3432`）→ 已登录，**跳过流程一**，直接进入流程二

> ⚠️ **登录弹窗结构**：登录弹窗有两个 Tab：**"手机号"** 和 **"邮箱"**，默认显示手机号 Tab。底部有分割线、**"使用 Google 登录"** 按钮、以及 **"我已同意条款和政策"** 复选框。还有 **"还没有账号？立即注册"** 链接，点击后打开独立的注册弹窗。

### 1.1 手机验证码登录

1. 打开首页 `/`，页面正常加载。
2. 点击导航栏右侧的 **"登录/注册"** 按钮。
3. 预期结果：弹出登录模态对话框，URL 路径**不变**（仍为 `/`），弹窗标题区域显示 **"欢迎来到 MindCloudX AI"**。
4. 确认当前 Tab 为 **"手机号"**（默认选中）。
5. **手机号输入**：登录弹窗使用 `vue-tel-input` 国际化手机号组件，左侧为国旗+区号下拉，右侧为手机号输入框。组件会根据用户 IP 自动检测国家（中文环境默认 CN `+86`，英文环境默认 US `+1`）。
   - 如果当前已是中国区号（`+86`），直接在输入框中输入手机号。
   - 如果需要切换区号：点击左侧国旗/区号下拉按钮，展开国家列表，在顶部搜索框中输入 `China`，从结果中找到并点击 **China**（`+86`）。
   - 在手机号输入框（placeholder 为 **"请输入手机号"**）中输入纯数字手机号 `13800138000`。
   - 组件使用 `international` 模式自动格式化（如 `+86 138 0013 8000`），内部的 `+86` 前缀是组件固有部分，不属于用户输入。
6. 预期结果：手机号格式校验通过，无红色错误提示出现。
7. **【测试阶段】填完手机号后，不要点击"获取验证码"按钮**，立即使用 `request_human_input` 操作弹窗请求测试者提供 4 位短信验证码（标题："请输入验证码"，说明："请查看手机短信并输入4位验证码"，占位符："验证码"）。测试者会自行在手机上获取验证码并输入。
8. 收到测试者输入的验证码后，将验证码填入验证码输入框（placeholder 为 **"验证码"**，`maxlength="4"`，`inputmode="numeric"`）。
9. 勾选 **"我已同意条款和政策"** 复选框（如果尚未勾选）。
10. 点击 **"登录/注册"** 按钮。
11. 预期结果：
    - 按钮显示加载状态，文字变为 **"登录中..."**。
    - 登录成功后弹出 toast 提示 **"登录/注册成功"**，弹窗自动关闭。
    - `localStorage` 中新增 `token` 字段（JWT 字符串）和 `user` 字段（JSON 对象）。
    - 导航栏 **"登录/注册"** 按钮消失，显示已登录状态（用户头像或昵称）。
    - URL 路径保持不变。

### 1.2 邮箱密码登录

1. 在登录弹窗中，点击 **"邮箱"** Tab 切换到邮箱登录表单。
2. 预期结果：显示邮箱输入框（placeholder 为 **"请输入邮箱"**，左侧有信封图标）和密码输入框（placeholder 为 **"请输入密码"**，右侧有显示/隐藏密码切换按钮）。
3. 在邮箱输入框中输入邮箱地址。
4. 在密码输入框中输入密码（6-32 位字符）。
5. 勾选 **"我已同意条款和政策"** 复选框（如果尚未勾选）。
6. 点击 **"登录/注册"** 按钮。
7. 预期结果：同 1.1 步骤 11。
8. 密码输入框下方有 **"忘记密码"** 链接，点击后可进入密码重置流程（传入当前邮箱）。

### 1.4 注册新账号

1. 在登录弹窗底部，点击 **"还没有账号？立即注册"** 链接。
2. 预期结果：弹出注册模态对话框，标题为 **"欢迎来到 MindCloudX AI"**，副标题显示注册相关文案。注册弹窗也有两个 Tab：**"手机号"** 和 **"邮箱"**。
3. **手机号注册**：填写手机号、验证码、昵称、密码。
4. **邮箱注册**：填写邮箱、邮箱验证码、昵称、密码。
5. 勾选条款复选框后点击注册按钮。
6. 预期结果：注册成功后自动登录，弹窗关闭。
7. 注册弹窗底部有 **"已有账号？立即登录"** 链接，点击后回到登录弹窗。

### 异常场景

- **手机号为空**：点击"获取验证码"或"登录/注册"时，手机号输入框下方显示红色提示 **"请输入手机号"**。
- **手机号格式不合法**：输入不符合国际格式的号码，失焦后显示红色提示 **"请输入有效的手机号码"**。
- **验证码为空**：点击"登录/注册"时，验证码字段显示红色提示 **"请输入验证码"**。
- **验证码格式错误**：输入非 4 位数字，显示红色提示 **"请输入4位验证码"**。
- **未勾选协议**：点击"登录/注册"时，复选框下方显示红色提示 **"请先同意条款和协议"**。
- **验证码错误**：提交后 API 返回错误，显示 toast 错误提示（服务端返回的 message）。
- **邮箱为空**：邮箱登录时，邮箱输入框失焦后显示红色提示 **"请输入邮箱"**。
- **邮箱格式不合法**：输入无效邮箱格式，失焦后显示红色提示 **"请输入有效的邮箱"**。
- **密码长度不足**：密码少于 6 位或多于 32 位，失焦后显示红色提示 **"请输入有效密码"**。
- **邮箱或密码错误**：邮箱登录提交后 API 返回错误，显示 toast **"邮箱或密码错误"**。
- **未登录访问受保护页面**：直接访问 `/datasets/list`，系统将 URL 存入 `sessionStorage`（key 为 `login_redirect`），然后跳转到 `/` 并自动弹出登录弹窗；若是页面内跳转则阻止导航并弹出登录弹窗。

## 流程二：数据集（项目）管理

**前置条件**：用户已登录。

> ⚠️ **重要限制**：`projectId=25` 的项目是生产数据，**禁止删除、禁止进入**。所有涉及删除、重命名、进入项目的操作，都必须避开该项目，选择其他项目操作。

### 2.1 查看项目列表

1. 点击导航栏中的 **"项目"** 菜单，或直接访问 `/datasets/list`。
2. 预期结果：浏览器标签页标题变为 **"项目 - MindCloudX AI"**。页面显示加载动画和文字 **"加载数据集..."**。
3. 加载完成后，显示项目卡片网格（响应式：1 列 / 2 列 / 3 列 / 4 列）。
4. 每张项目卡片上应展示以下信息：
   - 预览图（有 `cover_image` 则显示实际图，否则显示占位图）
   - 项目名称（可点击，点击进入训练任务列表）
   - 项目编码（前 6 位，带 ID 图标，点击可复制，复制后 toast 提示 **"复制成功"**）
   - 文件大小（格式化显示，如 `475.4 MB`）
   - 创建时间
   - 任务数量（格式：**"{0} 个任务"**）
   - 右上角 **"···"** 菜单按钮（Popover 弹出菜单，包含 **"重命名"** 和 **"删除"** 选项）
5. **搜索**：在搜索框（placeholder 为 **"搜索任务名称"**）中输入关键词，500ms 防抖后列表刷新，页码重置为 1。
6. **排序**：点击 **"排序方式"** 下拉菜单，可选 **"创建时间"** / **"名称"** / **"状态"**。点击排序方向按钮可切换升序/降序（图标在 `mdi:sort-ascending` 和 `mdi:sort-descending` 之间切换）。
7. **分页**：底部显示分页控件，每页 8 条。点击页码后列表刷新，URL query 参数中 `page` 同步更新。

### 2.2 上传数据集（创建项目）

> **[SKIP] 本节暂不测试** — "上传数据集"功能当前不可用，请跳过本节所有步骤，直接进入下一节。

   - 在 **"描述"** 文本域（placeholder 为 **"描述这个数据集的内容和用途"**）中填写描述（可选）。
   - 在 **"标签"** 输入框（placeholder 为 **"输入标签并按回车"**）中输入标签文字，按回车添加。每个标签右侧有 × 按钮可删除。
   - 勾选/取消 **"公开数据集（其他用户可以查看）"** 复选框。
   - 点击 **"下一步"** 按钮（数据集名称为空时按钮 disabled）。
4. **步骤二 — 上传图像**：
   - 页面显示两种上传方式：**"本地上传"** 和 **"远程URL"**，默认选中本地上传。
   - **本地上传**：将图像文件拖拽到虚线框区域（显示 **"拖放图像文件到此处，或"** 和 **"选择文件"** 按钮），或点击 **"选择文件"** 按钮选择文件。支持 JPG、JPEG、PNG 格式，可多选。

   - 页面标题 **"确认数据集信息"**，下方分两个区域展示摘要：
     - 基本信息区域：数据集名称、描述、可见性（**"公开"** 或 **"私有"**）、标签列表。
     - 图像文件区域：文件数量、总大小、相机参数文件名。
   - 底部提示文字：**"点击"开始上传"按钮后，系统将开始上传并处理您的数据集。上传完成后，您可以在数据集列表中查看并管理该数据集。"**
   - 点击 **"开始上传"** 按钮。
6. 预期结果：显示进度条和 **"上传中... X%"**，完成后通知提示 **"上传成功"** / **"数据集已成功上传并处理"**，自动跳转回 `/datasets`。

### 2.3 重命名项目

> ⚠️ 不要对 `projectId=25` 的项目执行此操作。

1. 在项目卡片信息区域，找到项目名称右侧的**菜单按钮**（`···`）。每个项目卡片**只有这一个按钮**。该按钮可能显示为：

   - `[按钮N/M @ 项目名]` 格式 — 选择**编号最大**的按钮

2. 预期结果：弹出 Popover 菜单，显示两个选项：**"重命名"** 和 **"删除"**。
3. 点击 **"重命名"**。
4. 预期结果：弹出对话框，标题为 **"重命名项目"**，标签为 **"项目名称"**，输入框（placeholder 为 **"请输入新的项目名称"**）中预填当前项目名称。底部两个按钮：**"取消"** 和 **"确认"**。

**前置条件**：用户已登录，已从项目列表进入训练页面，URL 中包含有效 `projectId`。

2. 预期结果：

   - 下方有 4 个 Tab 标签：**"点云"**、**"3DGS"**、**"CAD"**、**"Mesh"**。当前 Tab 由 URL 参数 `tab` 决定。
   - Tab 下方是工具栏（PC 端 `md:` 以上可见）：
