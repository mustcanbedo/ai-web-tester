## 流程一：用户登录（手机验证码登录）

**前置条件**：浏览器已清除 `localStorage` 中的 `token` 和 `user`。

1. 打开首页 `/`，页面正常加载。
2. 点击导航栏右侧的 **"登录/注册"** 按钮。
3. 预期结果：弹出登录模态对话框，URL 路径**不变**（仍为 `/`），弹窗标题区域显示 **"欢迎来到 MindCloudX AI"**。
4. **选择国家区号**：在手机号输入框左侧，点击国旗/区号下拉按钮，展开国家列表。
5. 在国家列表顶部的搜索框中输入 `China`，从结果中找到并点击 **China**（区号 `+86`）。
6. 预期结果：下拉收起，输入框左侧显示中国国旗和 `+86`。
7. **输入手机号**：在手机号输入框（placeholder 为 **"请输入手机号"**）中输入纯数字手机号 `13800138000`（不带 `+86` 前缀）。
8. 预期结果：手机号格式校验通过（中国手机号正则 `/^1[3-9]\d{9}$/`），无红色错误提示出现。
9. **【测试阶段】填完手机号后，不要点击"获取验证码"按钮**，立即使用 `request_human_input` 操作弹窗请求测试者提供 4 位短信验证码（标题："请输入验证码"，说明："请查看手机短信并输入4位验证码"，占位符："验证码"）。测试者会自行在手机上获取验证码并输入。
10. 收到测试者输入的验证码后，将验证码填入验证码输入框（placeholder 为 **"验证码"**，`maxlength="4"`，`inputmode="numeric"`）。
11. 勾选 **"我已同意条款和政策"** 复选框（如果尚未勾选）。
12. 点击 **"登录/注册"** 按钮。
13. 预期结果：
    - 按钮显示加载状态，文字变为 **"登录中..."**。
    - 登录成功后弹出 toast 提示 **"登录/注册成功"**，弹窗自动关闭。
    - `localStorage` 中新增 `token` 字段（JWT 字符串）和 `user` 字段（JSON 对象）。
    - 导航栏 **"登录/注册"** 按钮消失，显示已登录状态（用户头像或昵称）。
    - URL 路径保持不变。

### 异常场景

- **手机号为空**：点击"获取验证码"或"登录/注册"时，手机号输入框下方显示红色提示 **"请输入手机号"**。
- **手机号格式不合法**：输入非 11 位或不以 `1[3-9]` 开头的数字，失焦后显示红色提示 **"请输入有效的手机号码"**。
- **验证码为空**：点击"登录/注册"时，验证码字段显示红色提示 **"请输入验证码"**。
- **验证码格式错误**：输入非 4 位数字，显示红色提示 **"请输入4位验证码"**。
- **未勾选协议**：点击"登录/注册"时，复选框下方显示红色提示 **"请先同意条款和协议"**。
- **验证码错误**：提交后 API 返回错误，显示 toast 错误提示（服务端返回的 message）。
- **未登录访问受保护页面**：直接访问 `/datasets/list`，系统将 URL 存入 `sessionStorage`（key 为 `login_redirect`），然后跳转到 `/` 并自动弹出登录弹窗；若是页面内跳转则阻止导航并弹出登录弹窗。

## 流程二：数据集（项目）管理

**前置条件**：用户已登录。

> ⚠️ **重要限制**：`projectId=25` 的项目是生产数据，**禁止删除、禁止进入**。所有涉及删除、重命名、进入项目的操作，都必须避开该项目，选择其他项目操作。

### 2.1 查看项目列表

1. 点击导航栏中的 **"项目"** 菜单，或直接访问 `/datasets/list`。
2. 预期结果：浏览器标签页标题变为 **"项目 - MindCloudX AI"**。页面显示加载动画和文字 **"加载数据集..."**。
3. 加载完成后，显示项目卡片网格（响应式：1 列 / 2 列 / 3 列 / 4 列）。
4. 每张项目卡片上应展示以下信息：
   - 预览图（有 `cover_image` 则显示实际图，否则显示占位图）
   - 项目名称（可点击，点击进入训练任务列表）
   - 项目编码（前 6 位，带 ID 图标，点击可复制，复制后 toast 提示 **"复制成功"**）
   - 文件大小（格式化显示，如 `475.4 MB`）
   - 创建时间
   - 任务数量（格式：**"{0} 个任务"**）
   - 右上角 **"···"** 菜单按钮（Popover 弹出菜单，包含 **"重命名"** 和 **"删除"** 选项）
5. **搜索**：在搜索框（placeholder 为 **"搜索任务名称"**）中输入关键词，500ms 防抖后列表刷新，页码重置为 1。
6. **排序**：点击 **"排序方式"** 下拉菜单，可选 **"创建时间"** / **"名称"** / **"状态"**。点击排序方向按钮可切换升序/降序（图标在 `mdi:sort-ascending` 和 `mdi:sort-descending` 之间切换）。
7. **分页**：底部显示分页控件，每页 8 条。点击页码后列表刷新，URL query 参数中 `page` 同步更新。

### 2.2 上传数据集（创建项目）

> **[SKIP] 本节暂不测试** — "上传数据集"功能当前不可用，请跳过本节所有步骤，直接进入下一节。

2. 预期结果：页面标题显示 **"上传数据集"**，顶部有 3 步骤进度指示器（**"基本信息"** → **"上传图像"** → **"确认提交"**）。
3. **步骤一 — 基本信息**：
   - 在 **"数据集名称"**（必填，标有红色 `*`，placeholder 为 **"输入数据集名称"**）输入框中填写名称。
   - 在 **"描述"** 文本域（placeholder 为 **"描述这个数据集的内容和用途"**）中填写描述（可选）。
   - 在 **"标签"** 输入框（placeholder 为 **"输入标签并按回车"**）中输入标签文字，按回车添加。每个标签右侧有 × 按钮可删除。

   - 点击 **"下一步"** 按钮（数据集名称为空时按钮 disabled）。

6. 预期结果：显示进度条和 **"上传中... X%"**，完成后通知提示 **"上传成功"** / **"数据集已成功上传并处理"**，自动跳转回 `/datasets`。

## 流程三：训练任务管理

**前置条件**：用户已登录，已从项目列表进入训练页面，URL 中包含有效 `projectId`。

### 3.1 查看训练任务列表

1. 页面加载后，浏览器标签页标题变为对应的训练列表标题。
2. 预期结果：
   - 顶部显示数据集名称（来自 URL 参数 `datasetName`），左侧有 **"返回"** 按钮（点击回到 `/datasets/list`）。
   - 下方有 4 个 Tab 标签：**"点云"**、**"3DGS"**、**"CAD"**、**"Mesh"**。当前 Tab 由 URL 参数 `tab` 决定。
   - Tab 下方是工具栏（PC 端 `md:` 以上可见）：
     - **"创建训练"** 按钮（渐变色，左侧有 `+` 图标，`lg:` 以上可见）
     - 搜索框（placeholder 为 **"搜索任务名称"**）
     - 状态过滤下拉（标签 **"状态:"**，默认显示 **"全部"**，选项包括全部及各状态值）
     - 排序下拉（标签 **"排序:"**，选项为 **"创建时间"** / **"名称"** / **"状态"**）+ 升降序切换按钮

3. 切换 Tab 时，URL 参数 `tab` 同步更新，任务列表清空后重新加载对应类型数据。

7. 预期结果：

## 流程四：3D 模型查看与编辑

### 4.1 3DGS 模型编辑器

1. 在训练任务卡片上点击 **"编辑"** 按钮（仅任务创建者可见/可用）。
2. 预期结果：浏览器新标签页打开 `/gs/editor?taskId={taskCode}`，layout 为 `none`（无导航栏/侧边栏）。
3. 页面显示全屏加载动画。
4. 通过 `taskShareDetailApi` 获取任务详情。
5. 根据任务数据加载模型文件到 iframe 编辑器：
   - 优先加载 `ply_webp` 格式（非 Safari 浏览器使用 `meta.json`）。
   - 降级加载 `splat` 格式的 `result.splat`。
   - 如有 `model_config`，一并发送 `model_config.json` 给 iframe。
   - 否则如有 `extra_fields.colmap_path`，发送 `colmap.json` 给 iframe。
6. `editable` 属性取决于当前用户是否为任务创建者（`task.creator === userInfo.id`）。

8. 用户编辑模型配置后点击"发布配置"：
   - 若有封面图：通过 COS 临时密钥上传到 `public/cover_image/cover_{uuid}.png`。
   - 调用 `taskEditDetailApi` 保存 `model_config` 和 `cover_image`。
9. 预期结果：iframe 返回保存成功消息。

## 流程六：认证守卫验证

### 6.1 未登录访问受保护页面（首次访问/刷新）

1. 清除 `localStorage` 中的 `token`。
2. 直接在浏览器地址栏输入 `/datasets/list` 并回车。
3. 预期结果：
   - 页面跳转到 `/`（首页），`replace` 模式。
   - 自动弹出登录弹窗。
   - `sessionStorage` 中存入 `login_redirect` 值为 `/datasets/list`。

### 6.2 未登录页面内跳转

1. 清除 `localStorage` 中的 `token`。
2. 先访问 `/`，然后点击导航栏中的项目链接。
3. 预期结果：
   - 导航被阻止（`next(false)`），URL 保持为 `/`。
   - 弹出登录弹窗。

2. 预期结果：登录成功后自动跳转到 `/datasets/list`（从 `sessionStorage` 读取 `login_redirect`），`sessionStorage` 中 `login_redirect` 被清除。

### 6.4 Token 失效自动登出

1. 在已登录状态下，手动将 `localStorage` 中的 `token` 改为无效值。
2. 触发任意需要认证的 API 请求（如切换到项目列表页面）。
3. 预期结果：当 API 返回 `status: 999` 时，系统自动调用 `logout()`，清除 `localStorage` 中的 `token` 和 `user`。
